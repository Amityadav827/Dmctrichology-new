## Initial Risks / Blockers Before Installation

- Backend `package.json` does not include `mongoose`, `mongodb`, or `mongodb-memory-server`, but some scripts reference them.
- Local admin and website expect backend API at `http://localhost:10000/api`; backend default is `4000`, so local run should set `PORT=10000` for compatibility.
- Without real Supabase credentials, backend may start but Supabase-backed endpoints, auth, uploads, CMS data, forms, and dynamic content will fail.
- Without `JWT_SECRET`, login/register token creation and protected routes cannot work.
- Without email credentials, forgot-password email cannot work.
- Supabase Storage buckets `images` and `videos` must exist for uploads.
- `.mcp.json` points to a specific Supabase project ref; that is metadata, not a runnable credential.
